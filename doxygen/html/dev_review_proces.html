Code Review Process

Contributions to this project undergo a through review process to ensure code quality, consistency, and adhere to project standards. The review process helps maintain the integrity and realibility of the codebase. Follow these steps to submit your contribution for review.

  1. Create a Branch: Fork the repository and create a new branch for your changes. Use meaningful branch names such as 'feature/your-feature' or 'fix/issue-description; to describe the purpose of your changes. Keep the names relatively short.
  2. Commit Changes: Make commits that follow our conding standards and clude clear, descriptive messages. Don't commit with just 'a' over 70 times like I did in the past, it'll be a pain to rebase them later on.
  3. Testing: Testing is optional, but I highly recomended doing testing because it ensures that your contributions meet the project's quality standards and minimizes the risk of introducing issues into the codebase.
  4. Submit a Pull Request: Once your changes are ready, submit a pull request to the 'develop' branch. Provide a detailed description of your changes, explaining the problem you are solving and the approach taken to address it.
  5. Code Review: Your pull request will undergo review. I will provide feedback, suggestions and requests for changes. Respond promptly to address review comments.
  6. Approval and Merging: Once your pull request passes code review, it will be approved for merging into the 'develop' branch.
  7. Release Cycle: Changes merged into 'develop' are periodically merged into the 'main' branch during the release cycle.
