3. Database SQL

Database SQL

Database schema for this project consists of several key tables that store essential data for our application. Below is a summary of the main tables and their purposes:

  • Articles: Stores information for individual articles found on the blog page, including article IDs, article titles and title text.
  • Users: Stores user information, including user IDs, usernames, email addresses, and user roles.
  • Orders: Stores order information, including order IDs, pets adopted in that order, quantity, price, etc
  • Petsinfo: Stores information about individual pet, this includes pet ID, pet's name, age, adoption price, and species.
  • Petstories: Stores stories for individual pets, later to be displayed on their profiles, this includes ID of the pet, and their story.
  • Postcodes: Stores user's postcodes after completing orders.
  • Vets: Stores information about each veterinary staff in the clinic, including their personal details.
